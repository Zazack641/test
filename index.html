Scénario complet étape par étape pour créer un premier site web

  1. Créer un dépôt GitHub

    Objectif : avoir un "classeur" pour stocker le site.

    Étapes :

    1. Ouvrez github.com et connectez-vous.
    2. Cliquez sur New repository (nouveau dépôt).
    3. Donnez un nom simple, par ex. mon-premier-site.
    4. Choisissez Public.
    5. Cochez Add a README file (optionnel pour l’instant).
    6. Cliquez sur Create repository.

    Ainsi, vous aurez créé un endroit pour stocker votre site web, où tout sera sauvegardé et accessible publiquement.

  2. Ouvrir le dépôt dans Codespaces

    Objectif : coder directement dans le cloud, sans rien installer.

    Étapes :

    1. Dans votre dépôt GitHub, cliquez sur Code → Codespaces → Create codespace on main.
    2. Patientz quelques secondes : VS Code s’ouvre dans le navigateur.
    3. Explorez l’interface : à gauche, les fichiers ; en haut, le terminal.

    Vous travaillerez maintenant directement dans le cloud, sans avoir besoin d'installer quoi que ce soit sur votre ordinateur.

  3. Créer les fichiers de base

    Objectif : préparer la structure minimale du site.

    Étapes :

    1. Cliquez sur New File et créez index.html.
    2. Cliquez sur New File et créez style.css.
    3. Créez un dossier images (clic droit → New Folder).

    Vous séparez ainsi le contenu (HTML), la mise en forme (CSS) et les images, ce qui rend votre site facile à modifier.

  4. Utiliser l’IA pour générer la structure de base

    Objectif : gagner du temps et avoir un code propre.

    Prompt à donner à l’IA (copier-coller) :

    Agis comme un développeur web débutant mais rigoureux.
    Je veux créer un site web simple, clair et accessible.
    Contraintes :
    - site statique (HTML + CSS)
    - code lisible et commenté
    - pas de framework
    - adapté à un contexte scolaire
    Structure demandée :
    - index.html
    - style.css
    - dossier images/
    Éléments obligatoires :
    - en-tête avec un titre
    - menu simple
    - une section de contenu
    - pied de page
    Commence par créer les fichiers de base avec un contenu minimal fonctionnel.

    Vous commencez toujours par la structure de base. Le contenu et le design viendront ensuite.

  5. Visualiser le site depuis Codespaces

    Objectif : voir le résultat immédiatement.

    Étapes :

    1. Ouvrez le Terminal (en bas).
    2. Tape : python3 -m http.server
    3. Codespaces vous propose un lien Open in Browser → cliquez dessus.
    4. Votre site s’ouvre dans un nouvel onglet.

    Avant de publier, vous testez votre site localement. Vous pouvez ainsi modifier et actualiser instantanément.

  6. Modifier le site avec l’IA

    Objectif : enrichir le site étape par étape.

    Exemple de prompt "continuité" :

    À partir du site existant,
    modifie uniquement la section "Présentation" pour
    ajouter un texte plus court et plus clair.
    Ne change pas le reste du code.

    Vous guidez l'IA comme vous guideriez un élève : avec une consigne claire et précise à la fois.

  7. Publier le site sur GitHub Pages

    Objectif : rendre le site accessible sur Internet gratuitement via GitHub.

    Étapes :

    1. Retourne sur votre dépôt GitHub (github.com).
    2. Cliquez sur l'onglet "Settings" en haut à droite.
    3. Dans le menu de gauche, cliquez sur "Pages".
    4. Sous "Source", choisissez "Deploy from a branch".
    5. Sélectionnez la branche "main" et le dossier "/ (root)".
    6. Cliquez sur "Save".
    7. Attendez quelques minutes : une URL sera générée en haut de la page (ex : https://votre-utilisateur.github.io/mon-premier-site).

    GitHub héberge gratuitement votre site statique. Vous pouvez le partager avec cette URL.

  8. Vérifier et mettre à jour

    Objectif : maintenir le site à jour.

    Étapes :

    1. Retournez dans Codespaces.
    2. Modifie un fichier (par exemple, ajoute du texte dans index.html).
    3. Cliquez sur l'icône Source Control dans la barre latérale gauche (symbole de branche).
    4. Dans la section "Changes", cliquez sur le "+" à côté du fichier modifié pour le "stager".
    5. Entrez un message de commit dans le champ en haut (par exemple, "Ajout de modifications").
    6. Cliquez sur le bouton "Commit" (coche verte).
    7. Cliquez sur "Sync Changes" ou "Push" pour envoyer les changements sur GitHub.
    8. GitHub Pages met automatiquement le site à jour.

    Toutes vos modifications se reflètent automatiquement sur le site en ligne grâce à GitHub Pages.

  9. Points sécurité et bonnes pratiques

    Objectif : rester prudent et organisé dans la création de votre site.

    • Ne partagez pas de données sensibles (comme des noms ou photos personnelles) sur un site public.
    • Testez toujours votre site localement avant de le publier en ligne.
    • Gardez votre dépôt GitHub bien organisé avec des noms de fichiers clairs.
    • Rappelez-vous que GitHub Pages rend votre site accessible à tous sur Internet.

    Vous respectez ainsi les bonnes pratiques pour créer un site web simple et sécurisé.

Conclusion

En quelques étapes, vous pouvez créer, modifier, tester et publier un site web. Avec l’IA, vous vous concentrez sur vos idées plutôt que sur la technique. Le processus est reproductible et accessible à tous.
