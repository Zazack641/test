Scénario complet étape par étape pour créer un premier site web

  1. Créer un dépôt GitHub

    Objectif : avoir un "classeur" pour stocker le site.

    Étapes :

    1. Ouvre github.com et connecte-toi.
    2. Clique sur New repository (nouveau dépôt).
    3. Donne un nom simple, par ex. mon-premier-site.
    4. Choisis Public.
    5. Coche Add a README file (optionnel pour l’instant).
    6. Clique sur Create repository.

    Ainsi, vous aurez créé un endroit pour stocker votre site web, où tout sera sauvegardé et accessible publiquement.

  2. Ouvrir le dépôt dans Codespaces

    Objectif : coder directement dans le cloud, sans rien installer.

    Étapes :

    1. Dans ton dépôt GitHub, clique sur Code → Codespaces → Create codespace on main.
    2. Patiente quelques secondes : VS Code s’ouvre dans le navigateur.
    3. Explore l’interface : à gauche, les fichiers ; en haut, le terminal.

    Vous travaillerez maintenant directement dans le cloud, sans avoir besoin d'installer quoi que ce soit sur votre ordinateur.

  3. Créer les fichiers de base

    Objectif : préparer la structure minimale du site.

    Étapes :

    1. Clique sur New File et crée index.html.
    2. Clique sur New File et crée style.css.
    3. Crée un dossier images (clic droit → New Folder).

    Vous séparez ainsi le contenu (HTML), la mise en forme (CSS) et les images, ce qui rend votre site facile à modifier.

  4. Utiliser l’IA pour générer la structure de base

    Objectif : gagner du temps et avoir un code propre.

    Prompt à donner à l’IA (copier-coller) :

    Agis comme un développeur web débutant mais rigoureux.
    Je veux créer un site web simple, clair et accessible.
    Contraintes :
    - site statique (HTML + CSS)
    - code lisible et commenté
    - pas de framework
    - adapté à un contexte scolaire
    Structure demandée :
    - index.html
    - style.css
    - dossier images/
    Éléments obligatoires :
    - en-tête avec un titre
    - menu simple
    - une section de contenu
    - pied de page
    Commence par créer les fichiers de base avec un contenu minimal fonctionnel.

    Vous commencez toujours par la structure de base. Le contenu et le design viendront ensuite.

  5. Visualiser le site depuis Codespaces

    Objectif : voir le résultat immédiatement.

    Étapes :

    1. Ouvre le Terminal (en bas).
    2. Tape : python3 -m http.server
    3. Codespaces te propose un lien Open in Browser → clique dessus.
    4. Ton site s’ouvre dans un nouvel onglet.

    Avant de publier, vous testez votre site localement. Vous pouvez ainsi modifier et actualiser instantanément.

  6. Modifier le site avec l’IA

    Objectif : enrichir le site étape par étape.

    Exemple de prompt "continuité" :

    À partir du site existant,
    modifie uniquement la section "Présentation" pour
    ajouter un texte plus court et plus clair.
    Ne change pas le reste du code.

    Vous guidez l'IA comme vous guideriez un élève : avec une consigne claire et précise à la fois.

  7. Publier le site sur Netlify

    Objectif : rendre le site accessible sur Internet.

    Étapes :

    1. Ouvre netlify.com et crée un compte gratuit.
    2. Clique sur New site from Git → GitHub.
    3. Autorise Netlify à accéder à ton dépôt.
    4. Choisis le dépôt de ton site (mon-premier-site).
    5. Clique sur Deploy site.
    6. Une URL est générée automatiquement → partageable.

    GitHub stocke votre site, tandis que Netlify le rend accessible sur Internet. Votre dépôt peut rester privé si vous le souhaitez.

  8. Vérifier et mettre à jour

    Objectif : maintenir le site à jour.

    Étapes :

    1. Retourne dans Codespaces.
    2. Modifie un fichier (par exemple, ajoute du texte dans index.html).
    3. Clique sur l'icône Source Control dans la barre latérale gauche (symbole de branche).
    4. Dans la section "Changes", clique sur le "+" à côté du fichier modifié pour le "stager".
    5. Entre un message de commit dans le champ en haut (par exemple, "Ajout de modifications").
    6. Clique sur le bouton "Commit" (coche verte).
    7. Clique sur "Sync Changes" ou "Push" pour envoyer les changements sur GitHub.
    8. Netlify met automatiquement le site à jour.

    Toutes vos modifications se reflètent automatiquement sur le site en ligne grâce à cette intégration.

  9. Points sécurité et bonnes pratiques

    Objectif : rester prudent et organisé.

    • Ne pas mettre de données sensibles (ex : noms ou photos d’élèves).
    • Toujours tester avant de publier.
    • Garder le dépôt bien organisé.

    Vous séparez ainsi le contenu, le code et la publication pour rester simple et en sécurité.

Conclusion

En quelques étapes, on peut créer, modifier, tester et publier un site web. Avec l’IA, on se concentre sur nos idées plutôt que sur la technique. Le processus est reproductible et accessible à tous.
